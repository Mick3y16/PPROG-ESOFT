Exercício Jogadores Convocados

Pretende-se uma aplicação com interface gráfica para permitir à gestão de uma equipa de futebol criar um ficheiro de texto com a lista de jogadores convocados para um jogo. A interface gráfica deve visualizar duas listas, como é ilustrado na Figura 1. A lista da esquerda deve apresentar todos os jogadores do plantel e a lista da direita deve mostrar apenas os jogadores convocados para um jogo. O botão "Criar Ficheiro Convocados" deve gerar um ficheiro de texto contendo a lista de jogadores convocados. Os botões "Eliminar" devem remover o jogador selecionado da respetiva lista. O botão "Convocar Jogador" deve adicionar à lista de convocados, o jogador selecionado na lista completa. Esta operação só deve ser realizada para jogadores não convocados.

Figura 1 - Listas de Jogadores.

Os jogadores da lista completa devem ser introduzidos através da caixa de diálogo apresentada na Figura 2. Nesta caixa de diálogo, o utilizador insere o nome do jogador e escolhe, numa "Combo Box", a posição do jogador na equipa: "Guarda-Redes", "Defesa", "Médio" ou "Avançado". Esta caixa de diálogo deve ser visualizada selecionando a opção "Novo Jogador" do "Menu" mostrado na Figura 3.

Figura 2 - Caixa de diálogo para adicionar um novo jogador.

   
Figura 3 - Menu da aplicação.

O desenvolvimento desta aplicação deve satisfazer também os seguintes requisitos:
  • Partir do projeto base fornecido.
  • Utilizar uma arquitetura de duas camadas, separando o código da interface gráfica (camada ui) do código relacionado com os dados da aplicação (camada model).
  • A interface gráfica deve ser baseada em componentes "Swing".
  • As duas listas devem ser implementadas por objetos "JList" e que usam objetos "ListModel" para gerir os dados. Os modelos de dados devem ser do tipo "AbstractListModel".
  • A caixa de diálogo "Novo Jogador" deve ser do tipo "JDialog". Os dados devem ser validados e, no caso de serem inválidos, devem ser apresentadas mensagens apropriadas ao utilizador, através de caixas de diálogo do tipo "JOptionPane".
  • As duas listas de jogadores devem ser armazenadas num ficheiro binário, quando o utilizador pedir à aplicação para fazer o seguinte: fechar a janela, "Sair" ou "Guardar Listas". Os dados deste ficheiro devem ser carregados no arranque da aplicação. 
  • Para tornar a interface gráfica mais amigável, a opção do menu "Guardar Listas" e os botões de comando devem ser inibidos/desinibidos de forma adequada.
